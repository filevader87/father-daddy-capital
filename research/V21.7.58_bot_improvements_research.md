## 1. WORLD CUP BOT IMPROVEMENTS

### 1.1 Bivariate Poisson (vs Independent)
- Home/away goal correlation in WC: **-0.48 to -0.52** (negative — teams trade off scoring)
- Independent Poisson overestimates extreme results (both teams scoring 3+)
- Implementation: Gaussian copula or Karlis-Ntzoufras bivariate Poisson
- Expected improvement: ~0.5-1% AUC, critical for correct score markets
- **Priority: MEDIUM** — current overdispersion already partially addresses this

### 1.2 Covariates (ranked by impact)
1. **Tournament stage** (92% impact): Knockout → tighter defense, lower xG. Group → more open play
2. **Rest days** (63% impact): +0.08 xG per day above 7-day baseline, plateaus at 9 days
3. **Travel distance**: -0.04 xG per 1000km over baseline
4. **Altitude**: +0.1 xG per 500m elevation advantage
5. **Climate**: <10°C reduces scoring, >32°C reduces endurance
- **Priority: HIGH** — stage adjustment is easy to implement (group vs knockout flag)

### 1.3 Draw Probability Modeling
- Current: ad-hoc piecewise linear on ELO differential
- Better: Maher (1982) bivariate Poisson naturally produces draw probs
- Or: empirical draw inflation factor based on ELO gap
- WC draw rate: ~27% overall, 30% when ELO gap <50, 8% when gap >300
- **Priority: MEDIUM** — current model already blends ELO+Poisson 50/50

### 1.4 Ensemble Methods
- Recommended weights: 35% ELO+Poisson, 25% XGBoost, 25% logistic regression, 15% market
- XGBoost features: possession %, shot conversion, passing efficiency, H2H record
- Training data: WC 2014-2022 historical + recent internationals
- **Priority: LOW** — requires substantial data collection pipeline

### 1.5 Calibration
- Platt scaling: better for binary YES/NO markets
- Isotonic regression: handles non-monotonic calibration errors
- Temperature scaling: single-parameter, simplest to implement
- **Priority: HIGH** — can be trained on existing paper trade history

### 1.6 Group Stage Dead Rubbers
- Detect: team already qualified → rest starters → lower xG
- Adjust: reduce favorite xG by 15-20% if qualification secured
- Signal: group standings + remaining matches
- **Priority: HIGH** — WC 2026 group stage is ongoing

### 1.7 Closing Line Value (CLV)
- PM equivalent: compare entry price vs final settlement price
- Track: entry_price vs resolution_outcome (1.0 win / 0.0 loss)
- CLV = average(entry_price) - average(resolution_price) for winning trades
- Positive CLV = model has predictive edge beyond market
- **Priority: HIGH** — essential for measuring if model works

## 2. WEATHER BOT IMPROVEMENTS

### 2.1 Multi-Model Ensemble
- Optimal weights: NWS 25%, ECMWF 30%, GFS 15%, UKMET 20%, Canadian 10%
- Skill-based weighting via Brier score minimization
- Ensemble improves Brier score by 8-15% over single model
- Extreme event detection improves 30% at 95th percentile
- **Priority: HIGH** — ECMWF and UKMET data are free via open APIs

### 2.2 Calibration
- NWS probabilities are overconfident by 10-15% at extremes
- **Isotonic regression (PAVA)**: best for temperature distributions
- Platt scaling: better for binary YES/NO alignment
- Train per city-threshold pair on historical forecast vs actual
- **Priority: HIGHEST** — immediate Brier score improvement

### 2.3 Forecast Horizon
- 0-6h: LOW alpha (too rapid, resolution lag)
- 8-24h: MEDIUM alpha (sweet spot for 5-min polling)
- 48-72h: HIGH alpha (models diverge, market lags uncertainty)
- 5-10d: LOW alpha (meaningless, climatology only)
- **Priority: HIGH** — shift focus to 48-72h horizon markets

### 2.4 Temperature PDF (not point forecast)
- Model full distribution using ensemble spread + climatology
- Beta distribution for bounded [0, 35°C] summer temps
- Convert PDF to binary probability via CDF integration
- **Priority: MEDIUM** — requires ensemble data pipeline

### 2.5 Analog Forecasting
- Find top 5-10 historical patterns similar to current conditions
- Distance metric: Euclidean on normalized (temp, humidity, dewpoint)
- Window: same month ±2 days, ±48 hours
- Weight by inverse similarity distance
- **Priority: MEDIUM** — requires historical data archive

### 2.6 MOS (Model Output Statistics)
- Statistical post-processing of raw model output
- Linear regression: MOS_temp = β0 + Σβi * raw_forecast_vars
- Train per city using 2+ years of archived NWS forecasts
- **Priority: MEDIUM** — requires data collection pipeline

### 2.7 Extreme Event Handling
- Increase uncertainty inflation by 30% when |model - market| > 0.3
- Use KDE or parametric fit for tail probabilities
- Flag trades where model disagrees strongly with market for manual verification
- **Priority: HIGH** — prevents losses on miscalibrated extreme forecasts

### 2.8 Climatological Baseline
- 30-year NOAA normals reduce Brier score 6-10%
- ENSO phase: El Niño → +5% adjustment for tropical cities
- Download from NCEI: monthly mean ± SD per city
- **Priority: HIGH** — free data, immediate improvement

## 3. BTC SCALPER IMPROVEMENTS

### 3.1 Orderbook Imbalance (best 15m predictor)
- **Accuracy: 62-68%** edge over random walk using orderbook features
- Features: bid/ask depth ratio (top 15 levels), volume imbalance, spread
- Formula: RVI = (bid_depth - ask_depth) / (bid_depth + ask_depth)
- Threshold: |RVI| > 0.3 indicates directional pressure
- **Priority: HIGHEST** — already have WS orderbook data

### 3.2 VPIN (Volume-Synchronized Probability of Informed Trading)
- Measures toxicity of order flow
- High VPIN → informed flow → price movement imminent
- Implementation: bulk volume classification over 1-minute buckets
- VPIN > 0.3 = high toxicity, expect 15-30bp move
- **Priority: HIGH** — can be computed from existing WS trade data

### 3.3 Funding Rate Signal
- BTC perpetual funding rate predicts spot direction at 15-60m lag
- Funding > 0.05% (8h rate) → longs overlevered → mean reversion likely
- Funding < -0.03% → shorts squeezed → upward bias
- Data: Binance API (free), Coinglass API (free tier)
- **Priority: HIGH** — free data, strong signal

### 3.4 Liquidation Cascade Detection
- Precursors: rapid funding rate spike, large liquidation clusters, volume surge
- Coinglass API: free liquidation data, 1-minute granularity
- Signal: liquidations > $50M in 15m window → cascade → trend acceleration
- **Priority: MEDIUM** — requires Coinglass API integration

### 3.5 GARCH(1,1) Volatility Regime
- BTC 15m returns show volatility clustering (GARCH effects)
- Use GARCH to predict next-period volatility → adjust position sizing
- High vol regime → wider stops, smaller positions
- Low vol regime → tighter stops, larger positions
- **Priority: MEDIUM** — requires arch library

### 3.6 Data Sources
- **Binance WebSocket** (free): `wss://stream.binance.com:9443/ws/btcusdt@depth@1000ms`
- **Coinglass API** (free tier): `https://open-api-v3.coinglass.com/api/futures/liquidation/chart`
- **Binance Funding Rate** (free): `GET https://fapi.binance.com/fapi/v1/premiumIndex?symbol=BTCUSDT`
- **Priority: HIGH** — Binance WS depth is immediate upgrade

### 3.7 Stop-Loss for Binary Options
- Exit when: (a) market price moves against by >15pp, (b) model edge disappears, (c) time decay
- For 15m markets: exit at minute 10 if direction unclear (time value decay)
- Track CLV: if entry price > midpoint at minute 10, exit
- **Priority: HIGH** — critical for capital preservation

### 3.8 Portfolio Correlation Risk
- Multiple BTC PM bets = correlated exposure
- Limit: max 3 concurrent BTC-directional bets
- Hedge: if 3 BTC-YES positions, don't add 4th even if edge exists
- Net exposure cap: 50% of bankroll on single underlying
- **Priority: HIGH** — currently no correlation limit

## 4. TAIL RISK ENGINE IMPROVEMENTS

### 4.1 Identifying Mispriced Long-Shots
- Screen for: market_prob < 5% but model_prob > 15% (3x multiplier)
- Focus on: exchange hack, regulatory crackdown, ETF approval/rejection
- Crypto events with fat tails: depegs, bridge exploits, CEX insolvency
- **Priority: HIGH**

### 4.2 Kelly Sizing for Long-Shots
- Full Kelly on 5% prob with 30pp edge: f = (0.05+0.30 - 1) / (1-0.05) ≈ very small
- Use fractional Kelly (10%) for long-shots to survive variance
- Max position: 2% of bankroll per long-shot (not 5%)
- Need 20+ long-shot positions for law of large numbers
- **Priority: HIGH**

### 4.3 Stop-Loss for Tail Risk
- Exit if: market price drops 50% from entry (cut losses)
- Exit if: catalyst event passes without resolution (time decay)
- Roll profits: if position gains 3x, sell 50% to lock profit
- **Priority: MEDIUM**
